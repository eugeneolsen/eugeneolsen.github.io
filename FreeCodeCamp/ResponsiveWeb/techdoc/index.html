SMS with Amazon Web Services

using JavaScript and Node.js

by Eugene C. Olsen   16 January 2021
Introduction
Every week I get Short Message Service (SMS) messages (you know, text messages) on my phone from my banks, my brokerage firm, my cellular provider, my health insurer, and other big companies. Ever wonder how they do it? Amazon provides two Web services that will do the same thing: Amazon Simple Notification Service (SNS) and Amazon Pinpoint. If your business has at least one use case that would benefit by sending SMS messages to customer phones, you've come to the right place. Read on!
Prior Knowledge
  1. You should have an Amazon Web Services (AWS) account and be familiar with using the AWS console. Command line skills would be a plus. If you do not have an AWS account, Amazon will be happy to have you sign up for free and give you all sorts of great services free for 12 months. Sign up for a free Amazon Web Services account.
  2. You should be familiar with JavaScript. The AWS services speak other languages as well, but this documentation is written for JavaScript.
  3. Likewise, you should be familiar with Node.js.
Pricing

There are two classes of SMS messages for both SNS and Pinpoint: Transactional and Promotional. One part of the documentation says that Promotional pricing is less than Transactional pricing but the pricing tables show the same rates for both. This may change in the future.

In the United States, both Transactional and Promotional SMS messages cost $0.00645 using the Simple Notification Service (SNS). Inbound messages for SNS cost $0.0075. The Amazon Pinpoint service charges an additional $1.20 per 1000 endpoints per month.
